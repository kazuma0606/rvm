# エラーとトラブルシューティング

ForgeScript のエラーメッセージと対処法をまとめます。

## エラーの種類

ForgeScript では、次の3段階でエラーが発生する可能性があります。

1. **ファイル読み込みエラー** — 実行前
2. **構文エラー (Parse Error)** — 構文解析時
3. **コンパイルエラー (Compile Error)** — バイトコード生成時
4. **実行時エラー (Runtime Error)** — 実行時

## ファイル読み込みエラー

### Error reading file

**メッセージ例:**
```text
Error reading file 'program.fs': 指定されたパスが見つかりません。 (os error 2)
```

**原因:**
- 指定したファイルが存在しない
- パスが誤っている
- 権限がない

**対処法:**
- ファイルパスを確認する
- 相対パス・絶対パスを正しく指定する
- ファイルが存在するディレクトリで実行するか、正しいパスを指定する

```bash
# カレントディレクトリを確認
pwd   # Unix
cd    # Windows

# ファイルの存在確認
ls program.fs   # Unix
dir program.fs  # Windows
```

## 構文エラー (Parse Error)

構文解析時に不正なトークンや構造を検出した場合に発生します。

### Unexpected token

**メッセージ例:**
```text
Parse error: Unexpected token '=' at 3:5
```

**原因:**
- 文法に合わないトークンの出現
- キーワードの誤り

**対処法:**
- 表示された行・桁を確認する
- 該当箇所の文法を [言語リファレンス](language-reference.md) と照らし合わせる

**例:**
```fs
let x = 1
let y =     // 式が欠けている
```

### Unexpected end of input

**メッセージ例:**
```text
Parse error: Unexpected end of input at 2:7
```

**原因:**
- 式や文が途中で終わっている
- 括弧が閉じられていない

**対処法:**
- 式を完成させる
- `(`, `"` などの対応する閉じ括弧・引用符を追加する

**例:**
```fs
let x =     // 式が欠けている
let y = (1 + 2   // ) が欠けている
```

### Unexpected character

**メッセージ例:**
```text
Lex error: Unexpected character '#' at 1:1
```

**原因:**
- 未対応の文字（コメント記号など）の使用
- 不正な文字の混入

**対処法:**
- MVP ではコメント非対応のため、`#` や `//` は使わない
- 不正な文字を削除する

### Unclosed string

**メッセージ例:**
```text
Lex error: Unterminated string at 1:5
```

**原因:**
- 文字列リテラルの閉じ `"` が欠けている

**対処法:**
- 文字列の末尾に `"` を追加する

**例:**
```fs
let s = "Hello   // " が欠けている
```

## コンパイルエラー (Compile Error)

バイトコード生成時に、未定義変数などを検出した場合に発生します。

### Undefined variable

**メッセージ例:**
```text
Compile error: Undefined variable 'x' at 10:11
```

**原因:**
- 宣言前に変数を参照している
- 変数名の typo
- ネイティブ関数以外の未定義識別子の使用

**対処法:**
- 変数を先に `let` で宣言する
- 変数名のスペルを確認する
- ネイティブ関数（`print` など）は事前登録済みなので、通常はこのエラーにならない

**例:**
```fs
let y = x + 1   // x が未定義
let x = 10
```

正しい順序:
```fs
let x = 10
let y = x + 1
```

## 実行時エラー (Runtime Error)

プログラム実行中に発生するエラーです。

### Division by zero

**メッセージ例:**
```text
Runtime error: Division by zero
```

**原因:**
- 0 で除算している

**対処法:**
- 除数が 0 にならないようにする
- 除算前に除数をチェックする（現状は ForgeScript 側に分岐がないため、事前に値を確認する）

**例:**
```fs
let x = 0
let result = 10 / x   // エラー
```

### Type error

**メッセージ例:**
```text
Runtime error: Type error: expected Int, got String
```

**原因:**
- 算術演算に文字列を渡している
- 文字列連結に整数を渡している（現状は連結時に変換される可能性あり）
- 型が合わない演算

**対処法:**
- 演算のオペランドの型を確認する
- 必要に応じて適切な型に変換する（現状は変換機能が限定的）

### Undefined global

**メッセージ例:**
```text
Runtime error: Undefined global 'x'
```

**原因:**
- コンパイルを通過したが、実行時にグローバル変数が見つからない
- （通常はコンパイル時に検出されるが、特殊な経路で発生する可能性）

**対処法:**
- 変数が正しく宣言されているか確認する
- コンパイルエラーと同様に、宣言順序を確認する

### Stack underflow

**メッセージ例:**
```text
Runtime error: Stack underflow
```

**原因:**
- 内部的なスタック不整合（通常はコンパイラのバグ）

**対処法:**
- 問題の再現手順を記録し、バグ報告を検討する

### Not callable

**メッセージ例:**
```text
Runtime error: Not callable: Int
```

**原因:**
- 関数ではない値（整数や文字列など）を呼び出そうとしている

**対処法:**
- 呼び出し対象が関数（`print` など）であることを確認する
- 変数名の typo を確認する（`prin` ではなく `print` など）

**例:**
```fs
let x = 42
x()   // 整数は呼び出せない
```

## 終了コード

| 終了コード | 意味 |
|-----------|------|
| 0 | 正常終了 |
| 1 | 何らかのエラー（ファイル、構文、コンパイル、実行時） |

## デバッグのヒント

### 1. エラーの位置を確認する

エラーメッセージには `at 行:桁` のような位置情報が含まれることがあります。該当行を確認してください。

### 2. 小さく分割して試す

問題のコードを小さく分割し、REPL で1行ずつ実行して、どの部分でエラーになるか特定します。

```bash
forge
>>> let x = 1
>>> let y = x + 1
>>> print(y)
2
```

### 3. サンプルと比較する

[サンプルプログラム](examples.md) や [fixtures](../fixtures/) 内のファイルと比較して、文法の違いを確認します。

### 4. 言語リファレンスを参照する

[言語リファレンス](language-reference.md) で、使用している構文がサポートされているか確認します。
