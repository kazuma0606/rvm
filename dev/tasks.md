# RVM / ForgeScript Tasks

## 運用ルール

- タスク ID は工程番号ベースで `1-x`, `2-x` の形式で採番する
- 各実装タスクには必ず単体テストを含める
- 工程の節目ごとに結合テストを入れる
- CLI から一連のフローが通る段階で E2E を追加する
- 実装だけでなく、実行時チェックを各工程に入れる

## 実行時チェックの基本方針

各工程で最低限以下を入れる。

- `debug_assert!` または明示エラーで不変条件を確認する
- 不正入力時に panic ではなく診断可能なエラーを返す
- エラーにソース位置、命令位置、モジュール名などの文脈を付与する
- テストでは正常系だけでなく異常系も必ず確認する
